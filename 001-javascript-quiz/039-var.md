### JavaScript Hoisting with `var` Inside Loops

---

### Step 1: Given Code

	function foo() {
	  console.log(i)
	  for (var i = 0; i < 3; i++) {
	    console.log(i)
	  }
	}

	foo()

---

### Step 2: Core Concepts

---

### 2.1 var is Function Scoped (NOT Block Scoped)

- `var` does NOT respect `{}` blocks  
- It is scoped to the entire function  

So:

	var i inside for loop → belongs to function `foo`

---

### 2.2 Hoisting of var

- Declaration is moved to top of function  
- Initialized with `undefined`  

---

### Step 3: How JS Internally Sees It

	function foo() {
	  var i   // hoisted
	  console.log(i)
	  for (i = 0; i < 3; i++) {
	    console.log(i)
	  }
	}

---

### Step 4: Execution (Line by Line)

---

### First Line Inside Function

	console.log(i)

- `i` exists due to hoisting  
- Value = undefined  

Output:

	undefined

---

### for Loop Starts

	for (i = 0; i < 3; i++)

---

### Iteration 1

	i = 0  

	console.log(i)

Output:

	0

---

### Iteration 2

	i = 1  

	console.log(i)

Output:

	1

---

### Iteration 3

	i = 2  

	console.log(i)

Output:

	2

---

### Loop Ends

	i becomes 3 → condition fails  

---

### Step 5: Final Output

	undefined
	0
	1
	2

---

### Step 6: Key Insights

- var is hoisted to function scope  
- for-loop does NOT create new scope for var  
- variable exists before initialization → undefined  
- loop reuses same variable  

---

### Final Answer

	undefined
	0
	1
	2
